# <a name="getting-started-with-power-bi-gateways"></a>Power BI 게이트웨이 시작하기
**Power BI 게이트웨이 시작하기** 가이드입니다. 이 간략한 연습을 통해 게이트웨이의 역할 및 작동 방식과, 자체 게이트웨이 설치, 구성 및 실행 방법에 대해 학습합니다.  

![](media/service-gateway-getting-started/gw_gettingstarted_0a.png)

게이트웨이는 기술적인 주제이며 모든 네트워크와 기업이 각기 다르기 때문에 게이트웨이가 상당히 복잡할 수 있습니다. 복잡하지 않도록 기본적인 사항부터 시작하겠습니다.

## <a name="how-power-bi-gateways-work"></a>Power BI 게이트웨이 작동 방식
**게이트웨이**는 사설 온-프레미스 네트워크에 상주하는 데이터가 차후 Power BI 등의 클라우드 서비스에서 사용될 수 있게 데이터 액세스를 용이하게 하는 소프트웨어입니다. 연결 요청을 수신 대기하고 사용자의 요청이 특정 기준(예: 게이트웨이 사용 허가 여부)에 부합할 때만 액세스를 부여하는 게이트키퍼와 유사합니다. 이를 통해 조직에서는 데이터베이스와 웨어하우스를 온-프레미스 네트워크에 두면서도 해당 데이터의 하위 집합을 활용하여 안전하게 Power BI에서 강력한 보고서와 대시보드를 만들 수 있습니다.

게이트웨이도 자신을 통과하는 모든 데이터와, 데이터 원본에 연결하는 데 사용되는 모든 암호를 암호화 및 압축하여 데이터와 액세스를 보호합니다. 간단하게 들릴 수도 있지만 여기에서 여러 가지 사항을 고려해야 합니다.

때에 따라 전용 게이트웨이가 필요할 수 있습니다. 즉 대규모 Excel 통합 문서, 3개의 SQL 데이터베이스, 수년에 걸친 영업 및 마케팅 데이터가 있고 모든 관점에서 영업 데이터를 제시하는 Power BI 대시보드를 만들려 하는 경우가 있습니다. 사용자 혼자 보고서를 만들며 사용자 본인의 Excel 통합 문서이고 오직 사용자만 이 데이터베이스를 사용하여 Power BI 보고서를 만듭니다. 그러한 데이터 원본을 모두와 공유하는 것이 아닌 개인 전용 게이트웨이가 필요합니다.

또는 Analysis Services, SAP, Oracle, IBM 등, 다양한 공급업체의 데이터베이스와 기타 여러 다양한 데이터 원본을 사용하는 조직에서 많은 사람들이 데이터에 액세스하여 다양한 보고서를 만들게 할 수 있습니다. 이 경우 모든 원본에 대한 액세스를 구성하여 조직 내 여러 사람들과 공유할 수 있게 하는 게이트웨이가 필요합니다. 이것은 완전히 다른 종류의 게이트웨이입니다.

다행스럽게도 Power BI는 그러한 각각의 시나리오에 부합하는 두 가지 게이트웨이를 제공합니다. Power BI에서 제공하는 이러한 두 게이트웨이는 다음과 같습니다.

* **온-프레미스 데이터 게이트웨이(개인 모드)** – 한 사용자가 원본에 연결하며 다른 사용자와 공유할 수 없습니다. Power BI에만 사용할 수 있습니다.
* **온-프레미스 데이터 게이트웨이** – 여러 사용자가 여러 온-프레미스 데이터 원본에 연결할 수 있으며 단일 게이트웨이 설치로 Power BI, PowerApps, Flow 및 Azure Logic 앱에서 모두 사용할 수 있습니다.

두 게이트웨이 모두 유사한 기능을 수행합니다. 즉 개인 온-프레미스 네트워크에 상주하는 데이터에 액세스하여 데이터를 Power BI 같은 클라우드 기반 서비스에서 사용할 수 있게 합니다. 개인 게이트웨이는 한 사람이 Power BI를 통해서만 사용할 수 있으며 **온-프레미스 데이터 게이트웨이**는 많은 사용자와 많은 서비스에서 사용할 수 있습니다.

다음과 같은 세 부분 또는 단계를 통해 게이트웨이가 작동하게 됩니다.

* 게이트웨이 설치 
* 게이트웨이에 사용자 추가(게이트웨이를 사용할 수 있게 함)
* 데이터 원본 연결 

또한 게이트웨이를 사용하여 다음과 같이 중요할 수 있는 작업을 수행할 수 있습니다.

* Power BI 보고서를 새 데이터로 업데이트할 수 있게 온-프레미스 데이터를 새로 고칩니다.

데이터 새로 고침은 Power BI 대시보드와 보고서가 새롭게 되며 최신 데이터를 반영한다는 뜻입니다. 따라서 온-프레미스 데이터로 생성한 보고서를 누군가가 확인할 때 해당 보고서는 오래 전에 생성된 것이더라도 최신 정보를 사용할 수 있습니다.

첫 단계인 게이트웨이 설치는 간단합니다. 사용자가 게이트웨이에 액세스할 수 있게 하는 것도 간단합니다. Power BI 안에서 대화 상자에 추가하기만 하면 됩니다. 데이터 원본에 연결하는 것은 복잡할 수 있습니다. 데이터 원본이 다양하고 각기 자체의 연결 요구 사항과 미묘한 차이가 있기 때문입니다. 이 문서에서는 게이트웨이에 초점을 맞추기 위해 새로 고침은 다른 가이드에서 다루겠습니다.

쉬운 것부터 먼저 알아보는 차원에서 게이트웨이 설치를 살펴봅니다.

## <a name="install-the-gateway"></a>게이트웨이 설치 
게이트웨이를 설치하려면 Power BI 서비스(이 링크를 사용하여 브라우저에서 Power BI 서비스를 실행하고 로그온할 수 있음)를 열고 자신의 Power BI 계정으로 로그인합니다. Power BI 서비스에서 다음 그림에서처럼 오른쪽 상단 모서리의 **다운로드 아이콘**을 선택하고 **데이터 게이트웨이**를 선택합니다.

![](media/service-gateway-getting-started/gw_gettingstarted_01.png)

그러면 다운로드 페이지로 이동합니다. 여기서 **다운로드 게이트웨이** 버튼을 클릭하여 다운로드를 시작합니다.

![](media/service-gateway-getting-started/gw_gettingstarted_02.png)

이 화면에는 게이트웨이가 수행하는 작업에 관한 매우 간략한 설명이 있습니다. 또한 게이트웨이를 설치하면 실제로 설치를 수행한 컴퓨터에서 실행된다는 몇 가지 중요한 **경고**도 제공합니다. 또한 컴퓨터가 꺼지면 게이트웨이도 꺼집니다(실행되지 않으면 작동하지 않음). 무선 네트워크를 사용하는 컴퓨터에 설치하는 것도 최적은 아니므로 유선 네트워크에 연결된 컴퓨터를 사용해야 합니다.

준비되면 **다음**을 선택하여 설치를 계속합니다.

![](media/service-gateway-getting-started/gw_gettingstarted_03.png)

이제 게이트웨이를 온-프레미스 게이트웨이로 설치할지 또는 개인 게이트웨이로 설치할지 선택합니다. 이 가이드에서는 **온-프레미스 데이터 게이트웨이**를 설치합니다.

이 결정에서는 몇 가지 주의할 점이 있습니다.

* 두 게이트웨이에 모두 64비트 Windows 운영 체제 필요
* 도메인 컨트롤러에 게이트웨이를 설치할 수 없음
* 각 모드(개인 및 표준)에서 실행 중인, 동일한 컴퓨터에 최대 두 개의 온-프레미스 데이터 게이트웨이를 설치할 수 있습니다. 
* 동일한 컴퓨터의 동일한 모드에서 실행 중인 둘 이상의 게이트웨이를 사용할 수 없습니다.
* 여러 온-프레미스 데이터 게이트웨이를 서로 다른 컴퓨터에 설치하고 모두 동일한 Power BI Gateway 관리 인터페이스에서 관리할 수 있습니다(개인 제외, 다음 글머리 기호 참조).
* 각 Power BI 사용자에 대해 실행하는 개인 모드 게이트웨이는 하나만 사용할 수 있습니다. 다른 컴퓨터에도 동일한 사용자에 대한 다른 개인 모드 게이트웨이를 설치하는 경우 가장 최근 설치가 기존의 이전 설치를 대신합니다.

**다음**을 선택하면 게이트웨이 설치를 시작합니다. 설치 위치를 지정해야 하며 기본 위치를 사용하는 것이 가장 좋습니다.

![](media/service-gateway-getting-started/gw_gettingstarted_06.png)

설치 프로세스가 빠르게 진행되고 상태 표시줄이 나타납니다.

![](media/service-gateway-getting-started/gw_gettingstarted_06a.png)

거의 완료되면 게이트웨이에 사용할 계정을 확인해야 합니다. 이것은 Power BI에 로그인하는 데 사용하는 계정이어야 합니다(사용자 이름 및 암호). 게이트웨이는 Power BI 계정과 연결되며 Power BI 서비스 안에서 게이트웨이를 구성할 수 있습니다.

![](media/service-gateway-getting-started/gw_gettingstarted_07.png)

다음 그림에서처럼 로그인됩니다.

![](media/service-gateway-getting-started/gw_gettingstarted_08.png)

로그인 후에는 **복구 키**를 만들어야 합니다. 다른 문서에서 상세히 다룰 것이며 현 시점에서는 게이트웨이 복구 또는 이동에 필요하다는 점을 기억하세요.

![](media/service-gateway-getting-started/gw_gettingstarted_09.png)

모두 제대로 진행되면 게이트웨이를 사용할 준비가 되었다는 창이 표시됩니다.

![](media/service-gateway-getting-started/gw_gettingstarted_10.png)

온-프레미스 게이트웨이 설치가 완료되었습니다. 앞서 말한 것처럼 매우 간단한 프로세스입니다. 이제 다음 단계는 **사용자 추가** 또는 **데이터 원본을 추가**입니다. 최초 구성 후에는 둘 중 아무 것이나 먼저 추가하면 됩니다.

다음 섹션에서는 게이트웨이에 사용자를 추가한 다음 게이트웨이에 데이터를 추가하는 것을 다룹니다.

## <a name="add-users-to-a-gateway"></a>게이트웨이에 사용자 추가
이제 게이트웨이를 설치했으므로 **Power BI 서비스**에서 게이트웨이를 관리합니다. 게이트웨이 관리 화면을 열려면 Power BI 서비스에서 오른쪽 위 모서리에 있는 기어 아이콘을 선택하고 **게이트웨이 관리**를 선택합니다.

![](media/service-gateway-getting-started/gw_gettingstarted_15.png)

Power BI 서비스 캔버스 안에 게이트웨이를 관리할 수 있는 페이지가 표시됩니다. **게이트웨이 설정** 페이지는 다음과 같습니다.

![](media/service-gateway-getting-started/gw_gettingstarted_12.png)

**관리자**를 탭하거나 클릭하면 다음 관리자 관리 페이지가 표시됩니다. 이것은 어떤 사용자가 게이트웨이를 *관리*할 수 있는지를 의미하며, 게이트웨이의 해당 사용자가 각각의 개별 데이터 원본으로부터 추가(또는 제거)됩니다. 이것을 다음 몇 단락에서 살펴볼 것입니다.

![](media/service-gateway-getting-started/gw_gettingstarted_13.png)

데이터 원본을 설치하고 유효성을 확인(연결 성공)한 후에는 이 **게이트웨이 관리** 화면의 왼쪽에 있는 연결된 게이트웨이 아래 다음 그림처럼 표시됩니다. 오른쪽 창에는 **데이터 원본 설정** 및 **사용자** 등의 두 섹션이 있는데 둘 사이를 전환할 수 있습니다. 바로 다음 화면은 **데이터 원본 설정** 섹션입니다.

![](media/service-gateway-getting-started/gw_gettingstarted_16.png)

**사용자**를 선택하면 선택한 데이터 원본에 대해 액세스를 부여하려는 조직 내 사용자를 입력할 수 있는 텍스트 상자가 나타납니다. 다음 화면에서 보시듯이 저는 Maggie와 Adam을 추가했습니다.

전자 메일 주소를 텍스트 상자에 입력하기 시작하면 Power BI는 해당 이메일이 사용자가 입력 중인 내용과 일치하는 사용자의 목록을 표시하므로 해당 이름을 클릭하여 목록에 추가할 수 있습니다.

또한 전자 메일 그룹(별칭)으로 개인처럼 사용자 그룹 액세스를 허용할 수 있습니다.

![](media/service-gateway-getting-started/gw_gettingstarted_17.png)

**추가**를 선택하면 추가된 구성원이 상자에 표시되며, 원할 경우 더 추가할 수 있습니다. 사용자 제거도 간단합니다. 간단히 해당 이름 옆에 있는 확인란을 선택한 다음 상자 아래의 **제거** 버튼을 선택하면 됩니다.

![](media/service-gateway-getting-started/gw_gettingstarted_18.png)

매우 간단합니다. 액세스를 부여하려는 각각의 데이터 원본에 사용자를 추가해야 합니다. 각 데이터 원본에는 별도의 사용자 목록이 있고 각 데이터 원본마다 개별적으로 사용자를 추가해야 합니다.

## <a name="adding-data-sources"></a>데이터 원본 추가
물론 유용한 게이트웨이를 위해 데이터 원본을 추가할 수 있습니다. 이 부분에서 Power BI 게이트웨이의 복잡성이 문제가 됩니다. 즉 여러 다양한 데이터 원본을 사용할 수 있고 각기 고유의 요구 사항이 있습니다(종종 자체 필수 드라이버도 있음).

그러나 다른 글로 넘어가기 전에 여기서 데이터 원본 추가에 관해 살펴보겠습니다. **Power BI 서비스**의 **게이트웨이 관리** 페이지 안에서 데이터 원본을 추가할 게이트웨이를 선택하고 오른쪽 위 모서리에서 게이트웨이 목록 바로 위에 있는 **데이터 원본 추가**를 선택합니다.

이렇게 하면 **데이터 원본 설정** 창이 다음 그림에서처럼 오른쪽 창에 표시됩니다. 여기서 데이터 원본 이름을 지정하고(**데이터 원본 이름** 텍스트 상자에 입력) **데이터 원본 형식** 드롭다운 목록에서 형식을 선택합니다.

![](media/service-gateway-getting-started/gw_gettingstarted_14.png)

이제 게이트웨이가 설치되었으니 데이터 원본을 추가할 수 있습니다. 좋습니다! 데이터 원본에 관한 정보, 게이트웨이 사용에 대한 자세한 내용 및 기타 유용한 정보는 다음 섹션의 리소스를 참조하세요.

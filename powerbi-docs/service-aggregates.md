# <a name="aggregates-in-power-bi"></a>Power BI의 집계
## <a name="what-is-an-aggregate"></a>집계란?
열에 있는 행의 값을 수학적으로 결합하려는 경우가 있습니다. 수학 연산은 합계, 평균, 최대값, 개수 등이 될 수 있습니다. 열에 있는 행의 데이터 값을 결합하는 것을 집계라고 합니다. 해당 수학 연산의 결과는 *집계*입니다. 

숫자 필드는 일부 범주별 필드에 대해 집계할(예: 합계 또는 평균을 계산할) 값입니다.  예를 들어 "제품별 판매량" 및 "지역별 불량품 수"가 있습니다. 숫자 필드는 **측정값** 이라고도 합니다. 필드 목록에서 측정값은 ∑ 기호로 표시됩니다. 자세한 내용은 [보고서 편집기... 둘러보기](service-the-report-editor-take-a-tour.md)를 참조하세요.

때로는 측정값 은 실제로 계산된 측정값 입니다. Power BI의 계산된 측정값은 데이터(보고서의 기반이 되는 데이터 모델에 정의됨)와 함께 가져옵니다. 각 계산된 측정값에는 하드 코딩된 고유한 수식이 있습니다. 사용되는 집계는 변경할 수 없습니다. 예를 들어 합계인 경우 합계만 될 수 있습니다. 필드 목록에서 계산된 측정값 은 계산기 기호로 표시됩니다. 계산된 측정값을 생성하는 방법에 대한 자세한 내용은 [Power BI Desktop의 측정값](desktop-measures.md)을 참조하세요.

범주별 필드는 숫자 형식이 아니더라도 집계할 수 있습니다.  범주별 필드가 **값** 또는 **도구 설명** 등 *숫자 전용* 버킷에 배치되는 경우, Power BI는 각 범주의 발생 횟수를 계산하거나 각 범주의 고유 발생 횟수를 계산할 수 있습니다.  문자열 및 날짜의 경우 Power BI에는 가장 이른, 최근, 처음, 마지막 등 몇 가지 더 많은 집계 옵션이 있습니다.  

## <a name="why-dont-aggregates-work-the-way-i-want-them-to"></a>내가 원하는 방식으로 작업을 집계되지 않는 이유는 무엇입니까?
Power BI 서비스에서의 집계 작업은 혼동될 수 있습니다. Power BI가 숫자 필드의 집계를 변경하지 못하게 할 수 있습니다. 또는 연도와 같은 필드를 집계하지 않고, 발생 횟수만 계산하려는 경우가 있습니다.

대부분의 경우 Power BI 데이터 집합에서 필드를 분류하는 방식에 문제의 원인이 있습니다. 필드가 텍스트로 분류되어 합이나 평균을 계산할 수 없다고 할 수 있습니다. [데이터 집합 소유자만 필드를 분류하는 방식을 변경할 수 있습니다](desktop-measures.md).  

혼동을 피하는 방법은 이 문서의 끝에 **팁 및 문제 해결**이라는 특별 섹션에 설명되어 있습니다.  여기에서 답을 찾을 수 없는 경우 [Power BI 커뮤니티 포럼](http://community.powerbi.com)에 질문을 게시해주시면 Power BI 팀에서 신속하게 직접 답변해드릴 것입니다.

## <a name="change-how-a-numeric-field-is-aggregated"></a>숫자 필드가 집계되는 방식 변경
다른 지역의 매출 데이터의 합계를 구하는 차트가 있지만 평균값을 구하려는 경우를 가정해 보겠습니다. 

1. 보고서 편집용 보기에서 측정값을 시각화에 추가합니다.
2. 시각화 창에서 해당 필드를 찾은 다음 마우스 오른쪽 단추를 클릭하고 필요한 집계 유형을 선택합니다. 필요한 집계가 보이지 않으면 데이터 집합 소유자에게 문의합니다. 필드를 소유자별로 분류하는 방법에 문제가 있을 수 있습니다.  
   
   ![](media/service-aggregates/aggregate_new.png)
   
   > [!NOTE]
   > 드롭다운 목록에서 사용할 수 있는 옵션은 1) 선택된 필드와 2) 필드가 데이터 집합 소유자별로 분류된 방식에 따라 달라집니다.
   > 
   > 

필드 집계에 사용 가능한 일부 옵션:

* **요약 안 함**. 이 옵션을 선택할 경우 해당 필드의 각 값이 별도로 처리되고 요약되지 않습니다. 이 옵션은 일반적으로 요약하면 안 되는 숫자 ID 열이 있는 경우 사용합니다.
* **합계**. 해당 필드의 모든 값을 더합니다.
* **평균**. 값의 산술 평균을 구합니다.
* **최소값**. 가장 작은 값을 보여줍니다.
* **최대값**. 가장 큰 값을 보여줍니다.
* **개수(비어 있지 않음).** 비어 있지 않는 필드에 있는 값의 개수를 계산합니다.
* **개수(고유):** 해당 필드에서 다른 값의 개수를 계산합니다.
* **표준 편차.**
* **분산**.
* **중앙값**.  중앙값(중간값)을 표시합니다. 위와 아래에 동일한 개수의 항목이 있는 값입니다.  중앙값 두 개가 있는 경우 Power BI에서 평균을 계산합니다.

예를 들어, 이 데이터는

| 국가 | 금액 |
|:--- |:--- |
| 미국 |100 |
| 영국 |150 |
| 캐나다 |100 |
| 독일 |125 |
| 프랑스 | |
| 일본 |125 |
| 오스트레일리아 |150 |

다음 결과가 제공됩니다.

* **요약 안 함**: 각 값이 별도로 표시됩니다
* **합계**: 750
* **평균값**: 125
* **최대값**: 150
* **최소값**: 100
* **개수(비어 있지 않음):** 6
* **개수(고유):** 4
* **표준 편차:** 20.4124145...
* **분산:** 416.666...
* **중앙값:** 125

## <a name="use-a-non-aggregated-field-as-a-numeric-field"></a>집계되지 않은 필드를 숫자 필드로 사용
집계되지 않은 필드를 숫자 필드로 사용합니다. 예를 들어 제품 이름 필드가 있는 경우 값으로 추가한 다음 **개수** 또는 **고유 개수**로 설정할 수 있습니다. 

1. 예를 들어 **스토어 > 체인**을 선택한다고 가정합니다.
   
   ![](media/service-aggregates/count-of-chain-do_not_summarize.png)
2. 그리고 집계를 기본값인 **요약 안 함** 에서 **개수(고유)**로 변경하는 경우 Power BI는 다른 체인의 수를 계산합니다. 여기서는 Fashions Direct 및 Lindseys라는 두 개의 체인이 있습니다.
   
   ![](media/service-aggregates/aggregates_count.png)
3. 그리고 집계를 **개수**로 변경하면 Power BI는 총 수를 계산합니다. 여기서는 **체인**에 104개 항목이 있습니다. **체인** 을 필터로 추가하면 Fashions Direct에 37개 행, Lindseys에 67개 행이 있음을 알 수 있습니다.  
   
   ![](media/service-aggregates/count_of_chain_104.png)

## <a name="tips-and-troubleshooting"></a>팁 및 문제 해결
Q: **요약 안 함** 옵션이 없는 이유는 무엇입니까?

A: 선택한 필드가 계산된 측정값일 수 있습니다. 각 계산된 측정값에는 하드 코딩된 고유한 수식이 있음을 기억하세요. 계산을 변경할 수 없습니다.

Q: 내 필드**는** 숫자인데, **개수** 및 **고유 개수**와 같은 선택만 나타나는 이유는 무엇입니까?

A: 데이터 집합 소유자가 실수로 또는 의도적으로 필드를 숫자로 분류*하지* 않았다고 설명할 수 있습니다. 예를 들어 데이터 집합에 **연도** 필드가 있다면 **연도** 필드는 개수를 셈하지만(즉, 1974년도에 태어난 사람의 수) 합계나 평균을 계산하지 않을 가능성이 크기 때문에 데이터 집합 소유자는 이 필드를 텍스트로 분류할 수 있습니다. 소유자는 Power BI Desktop의 데이터 집합을 열고 **모델링** 탭을 사용하여 데이터 형식을 변경할 수 있습니다.  

A: 또 다른 원인으로는 범주별 값만 허용하는 버킷에 필드를 놓은 경우입니다.  이 경우 유일한 옵션은 개수 및 고유 개수입니다.

A: 세 번째 원인은 축용 필드를 사용하는 경우입니다. 예를 들어, 가로 막대형 차트 축에서 Power BI는 각 고유 값에 하나의 막대를 표시합니다. 이 경우 필드 값을 전혀 집계하지 않습니다. 

>[!NOTE]
>이 규칙의 예외는 분산형 차트로, X 및 Y축에 대해 집계된 값이 *필요*합니다.


Q: 분산형 다이어그램이 있고 내 필드를 집계*하지* 않고자 합니다.  어떻게 해야 합니까?

A: 필드를 X 또는 Y축 버킷이 아닌 **세부 정보** 버킷에 추가하세요.

Q: 숫자 필드를 시각화에 추가할 때 대부분의 기본값은 합계를 계산되지만 일부 기본값은 평균을 계산하거나 개수를 세거나 일부 다른 집계를 수행합니다.  기본 집계가 항상 동일하지 않는 이유는 무엇입니까?

A: 데이터 집합 소유자에게는 각 필드에 대한 기본 요약 정보를 설정하는 옵션이 있습니다. 데이터 집합 소유자인 경우 Power BI Desktop의 **모델링** 탭에서 기본 요약을 변경하세요.

Q: 내 필드**는** 숫자인데, 드롭다운 목록에 집계 옵션이 나타나지 않는 이유는 무엇입니까?

A: 필드에는 계산기 아이콘이 있는 경우, 이는 계산된 측정값이며 각각의 계산된 측정값에는 Power BI 서비스에서 변경할 수 없는 자체 하드 코딩된 수식이 있습니다. 사용 중인 계산은 평균이나 합계와 같은 간단한 집계일 수 있지만, "부모 범주에 대한 기여 비율"이나 "한 해 시작 이후 누계"와 같이 좀 복잡한 집계일 수도 있습니다. Power BI는 결과의 합계 또는 평균을 계산하지 않고 각 데이터 요소에 대해 하드 코딩된 수식을 사용하여 다시 계산하기만 합니다.

Q: 저는 데이터 집합 소유자이며 필드가 집계되지 않도록 하려고 합니다.

A: Power BI Desktop의 **모델링** 탭에서 **데이터 형식**을 **텍스트**로 설정합니다.

Q: 드롭다운의 옵션으로 **요약 안 함**이 나타나지 않습니다.

A: 필드를 제거하고 다시 추가해보세요.
